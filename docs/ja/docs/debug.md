# デバッグパネル

デバッグパネルは、ワークフローから離れることなく問題を診断して修正できるように、モバイルランタイムの可視性を提供します。

## 得られるもの

- セッション制御：開始、一時停止、続行、ステップ、停止
- ブレークポイント管理とクイック編集
- スレッドとコールスタックナビゲーション
- 変数とウォッチ検査
- 一時停止状態中の式の評価
- デバッグコンテキストのための統合出力ストリーム

## なぜ重要なのか

- バグ報告から根本原因までの時間が短くなる
- ツール間のコンテキスト切り替えが少なくなる
- 修正をプッシュする前の信頼性が向上する

## 典型的なモバイルデバッグフロー

1. ワークスペースからデバッグセッションを開始します。
2. ブレークポイントに到達し、スタック+変数を検査します。
3. 不確実な状態のためにウォッチ式を追加します。
4. 変更前に主要な式を評価します。
5. 修正を適用して動作を検証します。

## Timeline統合

デバッグイベントはTimelineに反映されるため、インシデント後のレビューと根本原因分析が接続されたままになります。
