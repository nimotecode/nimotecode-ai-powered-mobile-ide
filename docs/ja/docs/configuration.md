# 構成

このページは、AI、SSH、タスクワークフローに焦点を当てて、NimoteCodeを安定した日常使用のためにセットアップするのに役立ちます。

## AI構成戦略

少なくとも2つのプロファイルを作成します：

- 日常のコーディング用のプライマリプロファイル
- フェイルオーバーまたはコスト制御用のバックアッププロファイル

使用スタイルに合わせて各プロファイルを調整します：

- 正確なコードタスクには低いランダム性
- 大きなプロジェクト推論には高いコンテキスト
- 知覚される応答を速くするためにストリーミングを有効

## AIプロバイダーセットアップチェックリスト

- プロバイダー、APIキー、エンドポイント、モデルを追加する
- AIチャットで応答を検証する
- AIエージェントが選択したプロファイルで実行できることを確認する
- プロンプトと制限をコスト/パフォーマンス目標と一致させる
- APIキーはプラットフォームの安全なストレージを介して保存されます（例：iOS Keychain / Android Keystoreバックストレージ）

## SSH構成戦略

各サーバー環境に専用のプロファイルを保持します：

- 環境名（例：Staging、Production）
- ホスト、ポート、ユーザー、認証方式
- 予測可能なナビゲーションのための安定したワークスペースルートパス

推奨：

- 本番サーバーにはキーベースの認証を使用する
- 高リスク環境と低リスク環境を分離する
- タスクを実行する前にアクセス権を検証する
- パスワード認証を使用する場合は、セッション専用として扱う：パスワードはメモリ内で使用され、ローカルSSHプロファイルストレージには永続化されません

## Tasks構成戦略

繰り返しの結果に基づいてタスクを設計します：

- ビルドとテスト
- リントとフォーマット
- サービス再起動とヘルスチェック
- ログ検査ショートカット

ヒント：

- チームの慣習で明確なタスク名を使用する
- クイックフィルタリングのために関連するタスクをグループ化する
- 高リスクコマンドを明確にラベル付けする

## 起動準備完了ベースライン

日常使用前に確認します：

1. AIプロファイルがチャットとエージェントで動作する
2. SSHプロファイルが接続してワークスペースを開くことができる
3. 少なくとも1つのビルド/テストタスクが正常に実行される
4. Source Controlがコミットと同期を完了できる
