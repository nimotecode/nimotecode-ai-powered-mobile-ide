# SSHワークスペース

SSHワークスペースは、NimoteCodeをモバイルリモート開発コンソールに変えます。

## できること

- サーバー接続設定を保存して再利用する
- パスワードまたは秘密鍵方式で接続する
- リモートプロジェクトディレクトリで直接開発する
- リモートでファイルを閲覧、編集、作成、名前変更、削除する
- ターミナルとエディターを同じワークスペースコンテキストに保つ

## 典型的なリモートセッション

1. SSHプロファイルを選択します。
2. 接続してワークスペースのルートパスを開きます。
3. ファイルを編集し、ターミナルコマンドを実行します。
4. Source ControlとDebug/LSPを使用して検証とデリバリーを完了します。

## 推奨プラクティス

- 環境ごとにプロファイルを分割する
- 重要な環境では優先的に秘密鍵認証を使用する
- ワークスペースのルートパスを明確にし、誤操作のリスクを減らす
