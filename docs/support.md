# Support

Common questions and answers about NimoteCode.

## Getting Started

### How do I connect to a remote server?

1. Open the app and go to the SSH panel
2. Tap the "+" button to add a new connection
3. Enter your server details:
   - Host: Your server IP or domain
   - Port: 22 (default SSH port)
   - Username: Your server username
   - Authentication: Choose password or SSH key
4. Save and connect!

### What authentication methods are supported?

- **Password Authentication**: Simple username and password
- **SSH Key Authentication**: More secure key-based authentication

### Can I use NimoteCode without an internet connection?

Some features work offline (like the code editor), but remote development features require an internet connection.

## Features

### How many terminal sessions can I run?

You can run up to 6 concurrent terminal sessions at the same time.

### What programming languages are supported?

NimoteCode supports syntax highlighting for 50+ programming languages, including Python, JavaScript, Java, Swift, C/C++, Go, Rust, PHP, Ruby, and many more.

### How does the AI assistant work?

The AI assistant supports multiple providers:
- **Ollama**: Local AI (fastest, private)
- **OpenAI**: GPT models (most capable)
- **SiliconFlow**: Chinese-friendly cloud AI
- **Groq**: High-speed cloud AI

### Can I edit GitHub files?

Yes! With GitHub integration, you can:
- Browse your repositories
- Edit files directly
- Commit changes automatically

## Technical

### What are the system requirements?

**Android:**
- Android 8.0 or higher
- 2GB RAM minimum (4GB recommended)

**iOS:**
- iOS 14.0 or higher
- iPhone, iPad, or iPod Touch

### How large of files can I edit?

NimoteCode can handle files up to 100,000 lines.

### Is my data secure?

Yes! We use:
- Firebase authentication
- Encrypted SSH connections
- Secure token storage
- No data retention policy

## Troubleshooting

### SSH connection fails

- Check your internet connection
- Verify your server details
- Ensure SSH is enabled on your server
- Try different authentication method

### App crashes on launch

- Clear app cache
- Restart your device
- Update to the latest version
- Contact support if issue persists

### Terminal doesn't work

- Check your SSH connection
- Verify shell permissions
- Try a different terminal emulator
