# Панель Debug

Панель Debug дает вам видимость времени выполнения на мобильном устройстве, чтобы вы могли диагностировать и исправлять проблемы, не выходя из рабочего процесса.

## Что вы получаете

- Управление сессией: запуск, пауза, продолжение, шаг, остановка
- Управление точками останова и быстрое редактирование
- Навигация по потокам и стеку вызовов
- Проверка переменных и наблюдения
- Оценка выражений во время приостановленного состояния
- Интегрированный поток вывода для контекста отладки

## Почему это важно

- Более короткое время от отчета об ошибке до коренной причины
- Меньше переключений контекста между инструментами
- Лучшая уверенность перед отправкой исправлений

## Типичный поток отладки на мобильном устройстве

1. Запустите сессию отладки из вашего рабочего пространства.
2. Достигните точки останова и проверьте стек + переменные.
3. Добавьте выражения наблюдения для неопределенных состояний.
4. Оцените ключевые выражения перед изменениями.
5. Примените исправление и проверьте поведение.

## Интеграция с Timeline

События отладки отражаются в Timeline, поэтому анализ после инцидента и анализ коренной причины остаются связанными.
