# SSH-рабочая область

SSH-рабочая область превращает NimoteCode в мобильную консоль удаленной разработки.

## Что вы можете сделать

- Сохраняйте и повторно используйте конфигурации подключения к серверу
- Подключайтесь с использованием пароля или закрытого ключа
- Разрабатывайте напрямую в каталогах удаленных проектов
- Удаленно просматривайте, редактируйте, создавайте, переименовывайте и удаляйте файлы
- Поддерживайте терминал и редактор в одном контексте рабочей области

## Типичная удаленная сессия

1. Выберите профиль SSH.
2. Подключитесь и откройте корневой путь рабочей области.
3. Редактируйте файлы и выполняйте команды терминала.
4. Завершите проверку и доставку с помощью Source Control и Debug/LSP.

## Рекомендуемые практики

- Разделяйте профили по средам
- Предпочитайте аутентификацию по ключу для критических сред
- Уточняйте корневой путь рабочей области, чтобы снизить риск ошибок
