# Быстрый старт (3 минуты)

Это руководство оптимизировано для первого успеха за одну сессию.

## Цель

За 3 минуты вы сделаете следующее:

1. Откроете рабочее пространство
2. Запустите реальную команду
3. Отредактируете один файл
4. Используете одну панель высокой ценности (Source Control / AI / Debug)

## Шаг 1: Открыть рабочее пространство (30 секунд)

Выберите один режим:

- Local Explorer: открыть локальную директорию проекта
- Remote Explorer: подключиться через SSH

Если используете удаленный режим:

1. Создайте профиль SSH (`host`, `port`, `username`)
2. Выберите метод аутентификации (пароль или закрытый ключ)
3. Подключитесь и подтвердите корневой путь рабочего пространства

## Шаг 2: Запустить команду в терминале (30 секунд)

1. Откройте панель терминала
2. Запустите проверочную команду (например `pwd`, `ls` или вашу команду build/test)
3. Если вывод длинный, используйте поиск в терминале

## Шаг 3: Отредактировать и сохранить (45 секунд)

1. Откройте исходный файл из проводника
2. Отредактируйте одну строку
3. Сохраните файл
4. При необходимости используйте навигацию по символам или поиск/замену в файле

## Шаг 4: Выберите один путь результата (75 секунд)

### Путь A: Зафиксировать изменения

Откройте Source Control и:

- проверьте измененные файлы
- подготовьте изменения
- зафиксируйте (и отправьте, если настроено)

Ссылка: [/ru/docs/source-control](/ru/docs/source-control)

### Путь B: Попросить AI ускорить

Откройте панель AI и:

- выберите конфигурацию провайдера
- попросите объяснение/рефакторинг/предложения по исправлению
- при необходимости запустите AI Agent для автоматизации рабочего процесса

Ссылка: [/ru/docs/ai](/ru/docs/ai)

### Путь C: Диагностировать быстрее (Pro)

Откройте LSP/Debug и:

- проверьте диагностику
- примените действия с кодом
- запустите сессию отладки и проверьте переменные/наблюдение

Ссылки: [/ru/docs/lsp](/ru/docs/lsp), [/ru/docs/debug](/ru/docs/debug)

## Сценарии высокой ценности

1. Удаленный хотфикс на сервере, подобном производственному: `/ru/docs/ssh` + `/ru/docs/terminal` + `/ru/docs/source-control`
2. Быстрая сортировка ошибок: `/ru/docs/lsp` + `/ru/docs/debug` + `/ru/docs/timeline`
3. Автоматизация повторяющихся задач: `/ru/docs/tasks` + `/ru/docs/ai`
4. Стабильная ежедневная настройка: `/ru/docs/settings` + `/ru/docs/configuration`

## Следующий лучший шаг

После первого запуска сразу настройте это:

- Профиль провайдера AI: [/ru/docs/configuration](/ru/docs/configuration)
- Настройки редактора и клавиатуры: [/ru/docs/settings](/ru/docs/settings)
