# Source Control

Source Control은 모바일에서 변경 사항을 검토, 커밋, 동기화할 수 있도록 NimoteCode에 완전한 Git 워크플로우를 제공합니다.

## 할 수 있는 것

- 스테이징된, 스테이징되지 않은, 충돌된 파일 검토
- 일괄 또는 파일별로 변경 사항 스테이징 또는 취소
- pull/push/fetch로 커밋하고 동기화
- 브랜치와 stash 워크플로우 관리
- 커밋/파일/라인 기록과 차이점 검사

## Git AI 지원

내장 AI가 다음을 도울 수 있습니다:

- 커밋 메시지 생성
- 변경된 파일 설명
- 잠재적 변경 위험 강조

## 전형적인 커밋 흐름

1. 파일 차이점을 검토합니다.
2. 선택한 변경 사항을 스테이징합니다.
3. 커밋 메시지를 생성하거나 작성합니다.
4. 커밋하고 동기화합니다.
5. 대상 브랜치에서 검증합니다.

## 팀이 사용하는 이유

- 모바일에서 더 빠른 코드 배포
- 긴급 수정에 대한 마찰 감소
- AI 지원 검토로 더 나은 변경 품질
