# 디버그 패널

디버그 패널은 워크플로우를 벗어나지 않고 문제를 진단하고 수정할 수 있도록 모바일 런타임 가시성을 제공합니다.

## 얻을 수 있는 것

- 세션 제어: 시작, 일시 중지, 계속, 단계, 중지
- 중단점 관리 및 빠른 편집
- 스레드와 호출 스택 탐색
- 변수와 감시 검사
- 일시 중지 상태 중 표현식 평가
- 디버그 컨텍스트를 위한 통합 출력 스트림

## 중요한 이유

- 버그 보고서에서 근본 원인까지의 시간이 단축됩니다
- 도구 간 컨텍스트 전환이 적어집니다
- 수정을 푸시하기 전 신뢰성이 향상됩니다

## 전형적인 모바일 디버그 흐름

1. 워크스페이스에서 디버그 세션을 시작합니다.
2. 중단점에 도달하여 스택+변수를 검사합니다.
3. 불확실한 상태를 위해 감시 표현식을 추가합니다.
4. 변경 전에 주요 표현식을 평가합니다.
5. 수정을 적용하고 동작을 검증합니다.

## Timeline 통합

디버그 이벤트는 Timeline에 반영되므로 인시던트 후 검토와 근본 원인 분석이 연결된 상태로 유지됩니다.
