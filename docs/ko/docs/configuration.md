# 구성

이 페이지는 AI, SSH, 작업 워크플로우에 중점을 두고 NimoteCode를 안정적인 일일 사용으로 설정하는 데 도움이 됩니다.

## AI 구성 전략

적어도 두 개의 프로필을 만듭니다:

- 일일 코딩을 위한 기본 프로필
- 장애 조치 또는 비용 제어를 위한 백업 프로필

사용 스타일에 맞게 각 프로필을 조정합니다:

- 정확한 코드 작업에는 낮은 무작위성
- 큰 프로젝트 추론에는 높은 컨텍스트
- 인식된 응답을 더 빠르게 하기 위해 스트리밍 활성화

## AI 프로바이더 설정 체크리스트

- 프로바이더, API 키, 엔드포인트, 모델 추가
- AI 채팅에서 응답 확인
- AI 에이전트가 선택한 프로필로 실행할 수 있는지 확인
- 프롬프트와 제한을 비용/성능 목표와 정렬
- API 키는 플랫폼 보안 스토리지를 통해 저장됩니다 (예: iOS Keychain / Android Keystore 기반 스토리지)

## SSH 구성 전략

각 서버 환경에 전용 프로필을 유지합니다:

- 환경 이름 (예: Staging, Production)
- 호스트, 포트, 사용자, 인증 방법
- 예측 가능한 탐색을 위한 안정적인 워크스페이스 루트 경로

권장:

- 프로덕션 서버에는 키 기반 인증 사용
- 고위험 환경과 저위험 환경 분리
- 작업 실행 전에 액세스 권한 확인
- 비밀번호 인증을 사용하는 경우 세션 전용으로 취급: 비밀번호는 메모리 내에서 사용되며 로컬 SSH 프로필 스토리지에 유지되지 않습니다

## Tasks 구성 전략

반복적인 결과를 중심으로 작업을 설계합니다:

- 빌드 및 테스트
- 린트 및 포맷
- 서비스 재시작 및 상태 확인
- 로그 검사 단축키

팁:

- 팀 규칙에 따라 명확한 작업 이름 사용
- 빠른 필터링을 위해 관련 작업 그룹화
- 고위험 명령은 명확하게 레이블 지정

## 시작 준비 완료 기준선

일일 사용 전에 확인합니다:

1. AI 프로필이 채팅과 에이전트에서 작동하는지
2. SSH 프로필이 연결하고 워크스페이스를 열 수 있는지
3. 적어도 하나의 빌드/테스트 작업이 성공적으로 실행되는지
4. Source Control이 커밋과 동기화를 완료할 수 있는지
