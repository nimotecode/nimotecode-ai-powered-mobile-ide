# 常见问题 (FAQ)

查找关于 NimoteCode 的常见问题解答。

## 通用问题

### 什么是 NimoteCode？

NimoteCode 是一款跨平台移动 IDE，让您可以从手机或平板开发远程服务器。它结合了：

- **SSH/SFTP**：连接远程服务器
- **代码编辑器**：语法高亮编辑器
- **终端**：多个 SSH 终端
- **AI 助手**：AI 驱动的编程帮助
- **Git 集成**：版本控制
- **GitHub**：仓库管理

### 支持哪些平台？

- **Android**：Android 5.0 (API 21) 及以上
- **iOS**：iOS 12.0 及以上

### NimoteCode 是免费的吗？

NimoteCode 提供免费和付费计划：

**免费版**：
- SSH 连接
- 文件编辑
- 终端访问
- 基础 AI 功能

**专业版**：
- 多个 SSH 会话
- 高级 AI 功能
- Git 集成
- GitHub 集成
- 优先支持

### 可以离线使用 NimoteCode 吗？

有限的离线功能：
- 查看缓存的文件
- 编辑本地副本
- 基础编辑器功能

完整功能需要互联网连接：
- SSH 连接
- AI 功能
- GitHub 同步
- Git 操作

## SSH 和远程开发

### 如何连接到服务器？

1. 打开远程浏览器（📁）
2. 点击 **+ 添加连接**
3. 输入服务器详情：
   - 主机（IP 或域名）
   - 端口（默认：22）
   - 用户名
   - 密码或 SSH 密钥
4. 点击**连接**

### 可以使用 SSH 密钥吗？

可以！NimoteCode 支持 SSH 密钥认证：

1. 在连接设置中
2. 选择"SSH 密钥"认证
3. 粘贴您的私钥
4. 或从剪贴板导入

**支持的密钥类型**：
- RSA
- ED25519
- ECDSA

### 可以有多少个 SSH 连接？

- **免费版**：1 个并发连接
- **专业版**：无限保存的连接，6 个并发终端

### 为什么无法连接到服务器？

常见问题：

1. **凭证错误**：检查用户名/密码
2. **防火墙**：确保端口 22 开放
3. **SSH 服务**：验证 SSH 正在运行
4. **IP 地址**：检查 IP 是否更改
5. **密钥权限**：确保密钥格式正确

**解决方案**：
```bash
# 从计算机测试连接
ssh username@your-server.com

# 检查 SSH 服务
sudo systemctl status sshd
```

### 如何编辑服务器上的文件？

1. 通过 SSH 连接到服务器
2. 在文件浏览器中导航到文件
3. 点击打开
4. 使用语法高亮编辑
5. 更改在 5 秒后自动保存

### 可以上传/下载文件吗？

可以！使用 SFTP：

1. 长按文件或文件夹
2. 选择"下载"或"上传"
3. 选择目标位置

### 最大文件大小是多少？

- **推荐**：< 10 MB
- **最大**：100 MB
- **行数**：最多 100,000 行

大文件可能会被截断。

## 终端

### 如何打开终端？

1. 连接到 SSH 服务器
2. 点击底部栏中的终端图标
3. 终端在您的主目录中打开

### 可以有多少个终端？

- **最大**：6 个并发终端
- **每个连接**：每个 SSH 连接可以有多个终端

### 可以自定义终端颜色吗？

可以！前往：
1. 设置 → 终端
2. 选择颜色主题
3. 或自定义单个颜色

### 如何使用快捷命令？

快捷命令是预定义的命令：

1. 打开终端
2. 点击快捷命令按钮（⚡）
3. 选择要执行的命令

**默认命令**：
- `ls` - 列出文件
- `ls -la` - 列出所有文件
- `npm install` - 安装包
- `git status` - 检查 Git 状态
- `top` - 系统监控

### 可以添加自定义快捷命令吗？

目前，快捷命令是预定义的。自定义命令计划在未来更新中添加。

### 终端显示乱码？

这通常意味着：

1. **编码问题**：将终端设置为 UTF-8
2. **区域设置问题**：在服务器上设置区域
3. **字体问题**：使用等宽字体

**解决方案**：
```bash
# 在服务器上设置区域
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
```

## AI 助手

### 如何设置 AI？

1. 打开 AI 面板（🤖）
2. 点击"添加提供商"
3. 选择提供商：
   - Ollama（本地）
   - OpenAI
   - SiliconFlow
   - Groq
4. 输入 API 密钥或 URL
5. 保存配置

### 支持哪些 AI 提供商？

| 提供商 | 类型 | 速度 | 质量 |
|--------|------|------|------|
| Ollama | 本地 | 快 | 好 |
| OpenAI | 云端 | 中 | 优秀 |
| SiliconFlow | 云端 | 快 | 好 |
| Groq | 云端 | 很快 | 好 |

### 可以使用本地 AI 吗？

可以！使用 Ollama：

1. 在服务器上安装 Ollama
2. 拉取模型：`ollama pull llama2`
3. 启动 Ollama：`ollama serve`
4. 在 NimoteCode 中配置：
   - 提供商：Ollama
   - URL：http://your-server:11434
   - 模型：llama2

### 如何使用 AI 编辑代码？

1. 打开文件
2. 打开 AI 面板
3. 切换到"编辑"模式
4. 描述您想要更改的内容
5. AI 将生成代码更改
6. 审查并应用更改

### AI 不响应？

检查：
1. **API 密钥**：验证您的 API 密钥是否正确
2. **配额**：检查是否超出了配额
3. **互联网**：确保互联网连接
4. **URL**：验证 API 端点 URL
5. **模型**：检查模型名称是否正确

### 可以取消 AI 响应吗？

可以！在响应生成期间点击停止按钮（⏹️）。

## GitHub 集成

### 如何连接 GitHub？

1. 打开 GitHub 面板（🐙）
2. 点击"连接 GitHub"
3. 授权 NimoteCode
4. 已连接！

### 需要什么权限？

NimoteCode 请求：
- `repo`：访问您的仓库
- `user`：读取您的资料

### 可以使用 GitHub Enterprise 吗？

可以！配置：
1. 设置 → GitHub
2. 启用"GitHub Enterprise"
3. 输入您的企业 URL
4. 像往常一样连接

### 如何在 GitHub 上编辑文件？

1. 连接 GitHub
2. 浏览到文件
3. 点击打开
4. 编辑文件
5. 更改在 5 秒后自动提交

### 可以在 GitHub 上创建新文件吗？

可以：
1. 导航到目录
2. 点击 **+** 按钮
3. 输入文件名
4. 添加内容
5. 文件自动提交

1. 打开仓库
2. 点击分支选择器（顶部栏）
3. 选择不同的分支
4. 文件从新分支重新加载

## 故障排除

### 应用一直崩溃

尝试：
1. 清除应用缓存
2. 更新到最新版本
3. 重启设备
4. 重新安装应用

### 无法连接到服务器

1. 检查互联网连接
2. 验证服务器地址
3. 从计算机测试：`ssh user@server`
4. 检查防火墙设置
5. 尝试使用 SSH 密钥而不是密码

### 文件无法保存

1. 检查互联网连接
2. 验证文件权限
3. 检查服务器上的磁盘空间
4. 尝试手动保存

### 终端不工作

1. 确保 SSH 已连接
2. 检查 shell 是否可用
3. 重新连接到服务器
4. 重启应用

### AI 不响应

1. 检查 API 密钥
2. 验证互联网连接
3. 检查 API 配额
4. 尝试不同的提供商

### GitHub 无法加载

1. 检查互联网连接
2. 重新授权 GitHub
3. 清除应用缓存
4. 重新连接 GitHub

### Git 状态不显示

1. 确保目录是 Git 仓库
2. 检查服务器上是否安装了 Git
3. 验证 Git 已初始化：`git status`
4. 打开正确的目录

## 安全

### 我的数据安全吗？

是的！NimoteCode 使用：

- **加密 SSH**：所有 SSH 连接都是加密的
- **安全存储**：凭证安全存储
- **无数据收集**：我们不收集您的代码或数据
- **本地 AI**：使用 Ollama 完全本地化 AI

- **凭证**：在设备上本地加密
- **文件**：仅在您的远程服务器上
- **设置**：在设备上本地
- **AI 对话**：NimoteCode 不存储

### 可以使用 SSH 密钥吗？

可以！完全支持 SSH 密钥认证，推荐使用以获得更好的安全性。

### 如何撤销 GitHub 访问权限？

1. 前往 GitHub.com → Settings → Applications
2. 找到 NimoteCode
3. 点击"撤销访问权限"

## 性能

### 应用很慢

尝试：
1. 关闭未使用的终端
2. 清除应用缓存
3. 减少终端滚动缓冲
4. 使用较小的文件

### 大文件很慢

对于大文件：
1. 超过 100,000 行的文件会被截断
2. 考虑拆分大文件
3. 使用 `head` 或 `tail` 查看部分内容

### 终端卡顿

改善终端性能：
1. 在设置中减少滚动缓冲行数
2. 关闭未使用的终端
3. 避免非常长的输出命令
