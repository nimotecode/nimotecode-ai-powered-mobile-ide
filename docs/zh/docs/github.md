# GitHub 集成

NimoteCode 提供无缝的 GitHub 集成，让您可以直接从移动设备浏览、编辑和管理您的仓库。

## 概述

GitHub 集成功能：

- **仓库访问**：浏览所有仓库
- **文件编辑**：带语法高亮的编辑
- **提交更改**：自动提交编辑
- **分支支持**：使用不同分支工作
- **OAuth 认证**：安全的 GitHub 登录

## 认证

### 连接 GitHub

1. 打开 **GitHub** 面板
2. 点击**"连接 GitHub"**
3. 在 GitHub 上授权 NimoteCode
4. 连接成功！

### OAuth 流程

NimoteCode 使用 OAuth 进行安全认证：

1. 点击"连接 GitHub"
2. GitHub 授权页面打开
3. 授予 NimoteCode 权限
4. 自动重定向回来
5. 访问令牌安全存储

### 权限

NimoteCode 请求这些权限：

- **repo**：完整访问公开和私有仓库
- **user**：读取您的个人资料

### 断开连接

断开 GitHub 连接：

1. 打开**设置**
2. 找到 **GitHub** 部分
3. 点击**"断开连接"**
4. 确认断开

或在 GitHub 上撤销访问：
1. 转到 GitHub 设置 → 应用程序
2. 找到 NimoteCode
3. 点击"撤销访问"

## 仓库模式

### 单仓库模式

连接到特定仓库：

**配置**：
```yaml
仓库: owner/repo-name
分支: main (或您偏好的分支)
```

**功能**：
- 直接访问一个仓库
- 更快的加载
- 更简单的界面

### 多仓库模式

访问所有仓库：

**功能**：
- 列出所有仓库（公开 + 私有）
- 按字母顺序排序
- 显示仓库信息：
  - 描述
  - 公开/私有标识
  - 星标数

### 切换模式

1. 打开 GitHub 面板
2. 点击**设置**图标
3. 选择仓库模式
4. 配置仓库

## 浏览仓库

### 仓库列表

在多仓库模式下，您会看到：

```
┌─────────────────────────────────────┐
│ 🔒 my-private-repo                  │
│    私有 • 我的私有项目              │
├─────────────────────────────────────┤
│ 🌐 my-public-repo                   │
│    公开 • 开源项目                   │
├─────────────────────────────────────┤
│ 🔒 another-repo                     │
│    私有 • 另一个项目                 │
└─────────────────────────────────────┘
```

### 文件浏览器

导航仓库文件：

- **树形结构**：分层文件视图
- **懒加载**：展开时加载目录
- **文件图标**：不同文件类型显示不同图标
- **缓存内容**：缓存已加载的内容

### 文件信息

点击文件查看：

- **文件名**：带扩展名的名称
- **文件大小**：字节/KB/MB 大小
- **最后提交**：最新提交消息
- **SHA**：文件的 Git SHA

## 文件操作

### 查看文件

1. 导航到文件
2. 点击打开
3. 查看语法高亮

**支持的功能**：
- 50+ 种语言的语法高亮
- 行号
- 代码折叠
- 二进制文件的只读模式

### 编辑文件

1. 打开文件
2. 进行编辑
3. 5 秒后自动保存
4. 更改自动提交

**编辑功能**：
- 完整的文本编辑器
- 撤销/重做
- 查找和替换
- 语法高亮

### 创建文件

1. 导航到目标目录
2. 点击 **+** 按钮
3. 选择**"新建文件"**
4. 输入文件名
5. 编辑新文件
6. 自动提交创建文件

### 重命名文件

1. 长按文件
2. 选择**"重命名"**
3. 输入新名称
4. 确认重命名

**注意**：重命名会创建新文件并删除旧文件。

### 删除文件

1. 长按文件
2. 选择**"删除"**
3. 确认删除

**警告**：删除是永久性的，立即提交！

## 提交更改

### 自动提交

NimoteCode 自动提交更改：

- **触发**：最后一次编辑后 5 秒
- **提交消息**："Update filename.ext"
- **分支**：当前分支

### 手动提交

自定义提交消息：

1. 编辑文件
2. 点击**"提交"**按钮
3. 输入提交消息
4. 点击**"提交"**

### 提交历史

查看文件历史：

1. 打开文件
2. 点击**"历史"**按钮
3. 查看此文件的所有提交
4. 点击提交查看差异

## 分支支持

### 默认分支

在配置中设置默认分支：

```yaml
分支: main
```

### 分支显示

当前分支显示在标题栏：

```
my-repo (main)
```

### 切换分支

1. 打开仓库
2. 点击分支选择器
3. 选择不同分支
4. 文件从新分支重新加载

### 创建分支

目前，应用中不支持创建分支。使用 GitHub 网页界面或终端中的 Git 命令。

## 最佳实践

### 提交策略

- **小提交**：进行专注的小更改
- **清晰消息**：编写描述性的提交消息
- **提交前审查**：检查您的更改
- **测试更改**：提交前验证功能

### 文件管理

- **使用 .gitignore**：忽略生成的文件
- **避免大文件**：GitHub 有文件大小限制
- **二进制文件**：对二进制文件要谨慎
- **冲突意识**：注意合并冲突

### 安全

- **不要提交机密**：永远不要提交 API 密钥、密码
- **使用 .env 文件**：将机密保存在环境文件中
- **审查更改**：提交前始终审查
- **撤销访问**：不需要时断开连接

## 限制

### 文件大小限制

- **最大文件大小**：100 MB
- **仓库大小**：建议 < 1 GB
- **大文件**：对 > 50 MB 的文件使用 Git LFS

### 功能限制

- **分支创建**：应用中不支持
- **Pull 请求**：仅查看，不能创建
- **Issues**：仅查看，不能创建
- **合并冲突**：应用中无法解决

### 速率限制

GitHub API 有速率限制：

- **已认证**：5,000 请求/小时
- **未认证**：60 请求/小时

NimoteCode 积极缓存以最小化 API 调用。

## 故障排除

### 认证失败

1. 检查互联网连接
2. 尝试断开并重新连接
3. 在 GitHub 上撤销并重新授权
4. 检查 GitHub 状态页面

### 无法看到仓库

1. 验证仓库存在
2. 检查您的访问权限
3. 刷新仓库列表
4. 检查仓库是否私有

### 文件无法保存

1. 检查互联网连接
2. 验证写入权限
3. 检查文件大小限制
4. 尝试手动提交

### 速率限制

1. 等待限制重置（1 小时）
2. 减少 API 调用
3. 有效使用缓存
4. 如果持续，联系支持
