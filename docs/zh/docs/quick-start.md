## 连接到您的服务器

### 步骤 1：添加 SSH 连接

1. 点击左侧边栏中的**远程浏览器**图标（📁）
2. 点击 **+ 添加连接**
3. 输入服务器详情：

| 字段 | 描述 | 示例 |
|------|------|------|
| **名称** | 连接名称 | "我的服务器" |
| **主机** | 服务器 IP 或域名 | `192.168.1.100` 或 `server.example.com` |
| **端口** | SSH 端口 | `22`（默认） |
| **用户名** | SSH 用户名 | `root` 或 `ubuntu` |
| **认证类型** | 密码或 SSH 密钥 | 密码 |

### 步骤 2：连接

1. 点击**连接**
2. 如果使用密码认证，输入密码
3. 等待连接（通常 2-5 秒）

::: tip SSH 密钥
为了更好的安全性，建议使用 SSH 密钥认证而不是密码。
:::

### 步骤 3：浏览文件

连接后，您将看到服务器的文件系统：

- **点击**文件打开它
- **点击**文件夹展开它
- **长按**查看更多选项（重命名、删除等）

## 使用终端

### 打开终端

1. 连接到您的服务器
2. 点击底部栏中的**终端**图标
3. 终端在您的主目录中打开

### 运行命令

只需输入并按回车：

```bash
ls -la              # 列出文件
cd /var/www         # 切换目录
npm install         # 安装包
docker ps           # 列出容器
```

### 快捷命令

点击 ⚡ 按钮使用快捷命令：

- 📋 `ls` - 列出文件
- 📁 `ls -la` - 列出所有文件
- 📦 `npm install` - 安装包
- 🔀 `git status` - 检查 Git 状态
- 📊 `top` - 系统监控

### 多个终端

- 点击 **+** 打开新终端
- 点击终端标签切换
- 最大：6 个并发终端

## 使用 AI 助手

### 设置 AI

1. 点击左侧边栏中的 **AI** 图标（🤖）
2. 点击**添加提供商**
3. 选择您的提供商：

#### 选项 1：OpenAI（推荐）

```yaml
提供商: OpenAI
API 密钥: sk-xxxxxxxxxxxx
模型: gpt-4
```

在 [platform.openai.com](https://platform.openai.com) 获取您的 API 密钥

#### 选项 2：Ollama（免费且本地）

```yaml
提供商: Ollama
URL: http://your-server:11434
模型: llama2
```

在您的服务器上安装 Ollama：
```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama2
ollama serve
```

#### 选项 3：SiliconFlow（中国）

```yaml
提供商: SiliconFlow
API 密钥: sk-xxxxxxxxxxxx
模型: Qwen/Qwen2.5-7B-Instruct
```

在 [siliconflow.cn](https://siliconflow.cn) 获取您的 API 密钥

### 聊天模式

将 AI 用作编程助手：

1. 打开 AI 面板
2. 确保选择"聊天"模式
3. 输入您的问题
4. AI 将回复有用的信息

**示例**：
```
"如何修复这个错误：..."
"解释这段代码：..."
"最好的方法是..."
```

### 编辑模式

让 AI 编辑您的代码：

1. 打开文件
2. 打开 AI 面板
3. 切换到"编辑"模式
4. 描述您想要更改的内容
5. 审查 AI 的建议
6. 应用或拒绝更改

**示例**：
```
"为这个函数添加错误处理"
"重构这段代码使用 async/await"
"添加注释解释这段代码"
```

## 故障排除

### 无法连接到服务器

**检查**：
- ✅ 互联网连接
- ✅ 服务器地址正确
- ✅ SSH 端口开放（默认：22）
- ✅ 用户名和密码正确
- ✅ 服务器正在运行

**从计算机测试**：
```bash
ssh username@your-server.com
```

### 文件无法保存

**检查**：
- ✅ 互联网连接
- ✅ 服务器上的文件权限
- ✅ 服务器上的磁盘空间

### 终端不工作

**检查**：
- ✅ SSH 连接处于活动状态
- ✅ 服务器上有可用的 shell
- ✅ 尝试重新连接

### AI 不响应

**检查**：
- ✅ API 密钥正确
- ✅ 互联网连接
- ✅ API 配额未超出
- ✅ 模型名称正确
