# 调试面板

调试面板提供移动端运行时可视化能力，帮助你在同一工作流内完成定位与修复。

## 你将获得

- 会话控制：启动、暂停、继续、单步、停止
- 断点管理与快速编辑
- 线程与调用栈切换
- 变量与 Watch 检查
- 暂停态表达式求值
- 调试输出流统一查看

## 核心价值

- 缩短从报错到根因的路径
- 减少多工具来回切换
- 提升修复前验证信心

## 典型移动调试流程

1. 在工作区启动调试会话。
2. 命中断点后检查调用栈与变量。
3. 添加 Watch 观察关键状态。
4. 对关键表达式执行求值验证。
5. 应用修复并再次验证行为。

## 与 Timeline 联动

调试事件会进入 Timeline，便于复盘与根因分析。
