# 终端使用

NimoteCode 提供强大的终端体验，支持多会话、快捷命令和 AI 驱动的分析。

## 概述

NimoteCode 中的终端是一个功能完整的 SSH 终端，支持：

- **多会话**：最多 6 个并发终端
- **完整 Shell 支持**：Bash、Zsh、Fish 等
- **ANSI 颜色**：完整的颜色支持
- **Unicode**：完整的 Unicode 字符支持
- **AI 分析**：使用 AI 分析终端输出

## 打开终端

### 从 SSH 连接

1. 连接到 SSH 服务器
2. 点击工具栏中的**终端**图标
3. 新的终端会话自动打开

### 从文件浏览器

1. 长按任何文件或目录
2. 选择**"在终端中打开"**
3. 终端在该位置打开

### 多终端

- **新建终端**：点击终端面板中的 **+** 按钮
- **最大数量**：最多 6 个并发终端
- **标签界面**：每个终端有自己的标签

## 终端功能

### Shell 支持

NimoteCode 支持各种 shell 类型：

- **Bash**：最常见的 Linux shell
- **Zsh**：配合 Oh My Zsh 很受欢迎
- **Fish**：用户友好的 shell
- **Dash**：轻量级 shell
- **自定义 Shell**：服务器上安装的任何 shell

### 输入功能

- **命令历史**：使用 ↑↓ 方向键导航
- **Tab 补全**：按 Tab 键自动补全
- **复制/粘贴**：长按显示上下文菜单
- **文本选择**：长按并拖动

### 输出功能

- **ANSI 颜色**：完整的 256 色支持
- **ANSI 转义序列**：所有标准序列
- **Unicode**：完整的 Unicode 字符支持
- **Emoji**：完整的 emoji 支持

### 显示选项

- **字体大小**：在设置中调整
- **字体族**：从可用字体中选择
- **颜色主题**：选择终端配色方案
- **滚动缓冲**：最多 10,000 行历史

## AI 终端分析

使用 AI 分析终端输出，了解错误并获取建议。

### 如何使用

1. 在终端中运行命令
2. 等待输出完成
3. 点击 **AI 分析**按钮
4. 查看 AI 生成的分析

### 分析输出

AI 提供结构化分析：

```
**快速摘要：**
发生了什么的简要概述

**错误与警告：**
- 错误 1：描述和修复建议
- 错误 2：描述和修复建议

**输出摘要：**
从输出中提取的关键信息

**下一步：**
1. 推荐操作 1
2. 推荐操作 2
```

### 使用场景

- **错误诊断**：了解命令失败的原因
- **输出摘要**：快速获取长输出的摘要
- **修复建议**：获取 AI 驱动的修复建议
- **学习**：了解不熟悉的命令

### 支持的语言

AI 分析支持多种语言：

- 中文
- English
- 日本語
- 更多...

## 终端会话

### 会话管理

- **独立通道**：每个终端有唯一 ID
- **会话持久化**：切换面板时会话保持
- **自动清理**：关闭工作区时关闭会话
- **重新连接**：SSH 重连时会话重连

### 多会话

为不同任务使用多个终端：

```
终端 1: npm run dev     (开发服务器)
终端 2: git status      (版本控制)
终端 3: docker logs     (容器日志)
终端 4: vim config      (文件编辑)
```

### 会话最佳实践

1. **命名标签**：跟踪终端用途
2. **关闭未使用**：释放资源
3. **使用快捷命令**：加速常见任务
4. **分析错误**：使用 AI 分析进行故障排除

## 自定义

### 终端设置

从以下位置访问终端设置：

**设置 → 终端**

可用选项：

- **字体大小**：10-24pt
- **字体族**：等宽字体
- **颜色主题**：内置主题
- **滚动缓冲行数**：1000-10000
- **提示音**：启用/禁用
- **光标样式**：块状/下划线/竖线
- **光标闪烁**：启用/禁用

### 颜色主题

内置终端主题：

- **默认**：标准终端颜色
- **Solarized Dark**：流行的深色主题
- **Solarized Light**：流行的浅色主题
- **Dracula**：紫色调深色主题
- **Monokai**：经典编辑器主题
- **自定义**：定义自己的颜色

## 性能技巧

### 获得更好性能

1. **限制会话**：只保持需要的终端打开
2. **清除输出**：定期清除大量输出
3. **使用管道**：将大结果输出到文件
4. **关闭未使用**：关闭不使用的终端

### 处理大量输出

处理大量输出时：

```bash
# 输出到文件
command > output.txt

# 使用 less 查看
command | less

# 使用 head/tail 查看部分输出
command | head -n 100
command | tail -n 100
```

## 故障排除

### 终端无法打开

1. 检查 SSH 连接状态
2. 验证服务器上是否可用 shell
3. 尝试重新连接 SSH
4. 重启应用

### 显示问题

1. 检查终端设置
2. 验证字体兼容性
3. 尝试不同的颜色主题
4. 检查特殊字符

### 输入问题

1. 检查键盘设置
2. 验证终端模式
3. 尝试外部键盘
4. 检查卡住的键

### 性能问题

1. 减少滚动缓冲行数
2. 清除终端输出
3. 关闭未使用的终端
4. 检查服务器负载

## 高级功能

### 环境变量

在终端中设置环境变量：

```bash
export NODE_ENV=production
export API_KEY=your_key_here
```

### 命令别名

为常用命令创建别名：

```bash
alias ll='ls -la'
alias gs='git status'
alias npmd='npm run dev'
```

### Shell 脚本

运行 shell 脚本：

```bash
# 添加执行权限
chmod +x script.sh

# 运行
./script.sh
```
