# SSH 远程开发

NimoteCode 提供强大的 SSH 远程开发能力，让您可以从任何地方连接到服务器并编辑文件。

## 概述

SSH（安全外壳协议）是一种安全连接远程服务器的协议。NimoteCode 使用 SSH 提供：

- **安全连接**：加密连接到您的服务器
- **SFTP 文件管理器**：浏览和编辑远程文件
- **终端访问**：在远程服务器上运行命令
- **多会话支持**：同时使用多个服务器

## 连接管理

### 添加新连接

1. 打开**远程浏览器**面板
2. 点击 **+** 按钮或"添加连接"
3. 输入连接详情：
   - **名称**：连接的友好名称
   - **主机**：服务器 IP 地址或域名
   - **端口**：SSH 端口（默认：22）
   - **用户名**：SSH 用户名
   - **认证方式**：密码或 SSH 密钥

### 认证方式

#### 密码认证

1. 选择"密码"认证
2. 输入您的 SSH 密码
3. 点击"连接"

**注意**：密码安全存储在设备存储中。

#### SSH 密钥认证

1. 选择"SSH 密钥"认证
2. 粘贴您的私钥内容
3. 可选添加密码短语
4. 点击"连接"

**支持的密钥类型**：
- RSA
- ED25519
- ECDSA

### 连接功能

- **连接超时**：10 秒超时保护
- **自动重连**：断线后自动重连（最多 2 次）
- **连接状态**：实时状态显示
  - 🟢 已连接
  - 🟡 连接中
  - 🔴 已断开

### 管理连接

- **编辑**：修改连接设置
- **删除**：删除已保存的连接
- **复制**：创建现有连接的副本
- **快速连接**：点击即可连接

## 远程文件浏览器

### 浏览文件

连接后，您可以浏览远程文件系统：

- **树形视图**：分层文件和文件夹结构
- **文件图标**：不同文件类型显示不同图标
- **路径导航**：面包屑导航
- **刷新**：手动刷新目录内容

### 文件操作

#### 创建文件

1. 导航到目标目录
2. 点击 **+** 按钮
3. 选择"新建文件"
4. 输入文件名
5. 文件在编辑器中打开

#### 创建目录

1. 导航到父目录
2. 点击 **+** 按钮
3. 选择"新建文件夹"
4. 输入文件夹名

#### 重命名文件/文件夹

1. 长按文件或文件夹
2. 选择"重命名"
3. 输入新名称
4. 确认

#### 删除文件/文件夹

1. 长按文件或文件夹
2. 选择"删除"
3. 确认删除

**警告**：删除是永久性的，无法撤销！

### 文件编辑器

#### 打开文件

- 点击任何文件在编辑器中打开
- 文件加载时自动语法高亮
- 大文件（>100,000 行）会被截断

#### 编辑文件

- **语法高亮**：根据文件扩展名自动识别
- **自动保存**：5 秒后自动保存更改
- **撤销/重做**：多级历史记录
- **行号**：可选的行号显示

#### 保存文件

- **自动保存**：默认启用（5 秒延迟）
- **手动保存**：点击保存按钮或使用快捷键
- **保存状态**：保存进度的视觉指示器

## 终端会话

### 打开终端

1. 连接到 SSH 服务器
2. 点击**终端**图标
3. 新的终端会话打开

### 多终端支持

- **并发会话**：最多同时 6 个终端
- **标签界面**：每个终端有自己的标签
- **会话持久化**：切换面板时会话保持
- **独立通道**：每个终端有唯一的通道 ID

### 终端功能

- **完整 Shell 支持**：Bash、Zsh、Fish 等
- **命令历史**：使用方向键导航历史
- **Tab 补全**：按 Tab 键自动补全
- **ANSI 颜色**：完整颜色支持
- **Unicode 支持**：完整的 Unicode 字符支持

### 终端控制

- **新建终端**：打开额外的终端会话
- **关闭终端**：关闭当前终端
- **清空终端**：清除终端内容
- **全屏**：将终端扩展到全屏

## AI 终端分析

NimoteCode 可以使用 AI 分析终端输出：

### 如何使用

1. 在终端中运行命令
2. 点击 **AI 分析**按钮
3. AI 分析输出并提供：
   - 快速摘要
   - 错误识别
   - 修复建议
   - 下一步操作

### 分析格式

```
**快速摘要：** 输出的简要概述

**错误与警告：** 错误列表和修复建议

**输出摘要：** 输出中的关键信息

**下一步：** 推荐的操作
```

- **密钥密码短语**：保护您的私钥
- **定期更新**：保持密钥更新
- **最小权限**：使用最小权限原则

### 性能

- **大文件**：对非常大的文件要谨慎
- **网络质量**：良好的连接改善体验
- **终端会话**：关闭未使用的终端
- **文件缓存**：需要时刷新

### 工作流程

1. **频繁保存**：让自动保存为您工作
2. **使用书签**：标记经常访问的目录
3. **多终端**：为不同任务使用不同终端
4. **快捷命令**：利用快捷命令提高效率

## 故障排除

### 连接失败

1. 检查网络连接
2. 验证服务器地址和端口
3. 确认用户名和凭据
4. 检查防火墙设置
5. 尝试使用密码代替密钥（或反之）

### 文件无法加载

1. 检查文件大小（可能太大）
2. 验证文件权限
3. 检查连接稳定性
4. 尝试刷新目录

### 终端不工作

1. 确保 SSH 连接处于活动状态
2. 检查服务器上是否可用 shell
3. 尝试关闭并重新打开终端
4. 如需要，重启应用

### 性能缓慢

1. 检查网络延迟
2. 减少打开的终端数量
3. 关闭未使用的文件标签
4. 检查服务器负载
